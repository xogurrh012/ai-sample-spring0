# 기능 명세서 (Functional Specification)

## 1. 개요
본 문서는 "심플 다중 사용자 블로그" 서비스의 구체적인 기능 사양을 정의합니다. 모든 기능은 REST API 기반으로 설계되며, 사용자 경험(UX)은 단순함과 직관성을 최우선으로 합니다.

## 2. 도메인별 상세 기능

### 2.1 회원 (User)
| 기능 ID | 기능명 | 상세 설명 | 비고 |
|:---:|:---|:---|:---|
| US-01 | 회원가입 | 아이디, 비밀번호, 이름을 입력받아 계정을 생성한다. | 중복 아이디 체크 포함 |
| US-02 | 로그인 | 아이디/비밀번호 인증 후 JWT 토큰을 발급한다. | - |
| US-03 | 회원 탈퇴 | 계정을 삭제하며, 해당 사용자가 작성한 모든 게시글/댓글을 논리적 삭제 처리한다. | - |

### 2.2 게시글 (Board)
| 기능 ID | 기능명 | 상세 설명 | 비고 |
|:---:|:---|:---|:---|
| BR-01 | 목록 조회 | 전체 게시글을 최신순으로 조회한다. (한 페이지당 N개) | 비회원 허용 |
| BR-02 | 상세 조회 | 특정 게시글의 제목, 본문, 작성자, 작성일, 댓글 목록을 조회한다. | 비회원 허용 |
| BR-03 | 글 작성 | 로그인한 사용자가 제목과 본문(텍스트 전용)을 입력하여 글을 저장한다. | 회원 전용 |
| BR-04 | 글 수정 | 본인이 작성한 글의 제목 또는 본문을 수정한다. | 회원 전용 |
| BR-05 | 글 삭제 | 본인이 작성한 글을 삭제한다. (연관된 댓글도 함께 삭제 처리 권장) | 회원 전용 |
| BR-06 | 글 검색 | 제목 또는 본문에서 특정 키워드가 포함된 글 목록을 필터링한다. | 비회원 허용 |

### 2.3 댓글 (Reply)
| 기능 ID | 기능명 | 상세 설명 | 비고 |
|:---:|:---|:---|:---|
| RP-01 | 댓글 작성 | 특정 게시글에 텍스트 기반의 댓글을 작성한다. | 회원 전용 |
| RP-02 | 댓글 삭제 | 본인이 작성한 댓글을 삭제한다. | 회원 전용 |

## 3. 공통 규칙 및 제약 사항
- **권한 처리**: 
    - 조회(Read) 계열은 비회원에게도 개방한다.
    - 생성(Create), 수정(Update), 삭제(Delete)는 반드시 유효한 JWT 토큰이 필요하다.
    - 수정/삭제 시에는 해당 리소스의 소유자(작성자)와 요청자가 동일한지 서버 측 검증을 수행한다.
- **데이터 정합성**:
    - 게시글 삭제 시 해당 글에 달린 모든 댓글은 함께 삭제 처리되어야 한다.
    - 탈퇴한 사용자의 이름은 '탈퇴한 사용자' 등으로 표시하거나 목록에서 완전히 제외한다. (PRD에 따라 비노출 우선)

## 4. 예외 상황 처리
- 존재하지 않는 게시글/댓글 요청 시: 404 Not Found 응답.
- 본인 소유가 아닌 리소스 수정/삭제 시도 시: 403 Forbidden 응답.
- 인증되지 않은 사용자가 권한 필요 기능 접근 시: 401 Unauthorized 응답.
