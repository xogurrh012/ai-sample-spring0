이 스킬은 코드 작성 전 단계별 승인을 거쳐 안전하고 투명하게 작업을 진행하기 위한 워크플로우를 정의합니다.

## 핵심 워크플로우 (Core Workflow)

1. **질문 및 의도 파악**: 트리거 문구 수신 시, 가장 먼저 **"어떤 계획을 세울거에요?"**라고 질문하여 사용자로부터 작업 대상을 확인받습니다.
2. **작업 제안 (Proposal)**: 사용자가 전달한 계획을 바탕으로, AI가 이 계획을 어떻게 수행할 것인지(수정할 파일, 로직 등)에 대한 상세 작업 보고서를 화면에 출력합니다. 만약에 프론트앤드/화면 페이지 개발이 포함된다면 아래의 예시와 같이 어떻게 화면을 그리는지 표시해줘. 아래는 예시야!
   1 +---------------------------------------------------------+
   2 | [Header Navbar] (기존 레이아웃) |
   3 +---------------------------------------------------------+
   4 | |
   5 | +-------------------------------------------+ |
   6 | | 회원가입 | |
   7 | |-------------------------------------------| |
   8 | | | |
   9 | | 아이디 | |
   10 | | [ input: username ] [ 중복체크 버튼 ] | |
   11 | | \* 메시지: (사용 가능한 아이디입니다) | |
   12 | | | |
   13 | | 비밀번호 | |
   14 | | [ input: password ] | |
   15 | | | |
   16 | | 이메일 | |
   17 | | [ input: email ] | |
   18 | | | |
   19 | | [ 주소 관련 입력 필드 (다음 단계 대비) ] | |
   20 | | | |
   21 | | [ 가입하기 버튼 ] | |
   22 | +-------------------------------------------+ |
   23 | |
   24 +---------------------------------------------------------+
   25 | [Footer] (기존 레이아웃) |
   26 +---------------------------------------------------------+
3. **사용자 승인 (Approval)**: 사용자가 계획을 검토하고 승인 여부를 결정합니다. 승인되지 않으면 작업을 진행하지 않고 계획을 수정하거나 다시 세웁니다.
4. **계획 실행 (Execution)**: 사용자의 최종 승인이 떨어진 후에만 실제 코드 수정 및 기능을 구현합니다.
5. **사후 보고서 작성 (Post-Execution Report)**: 모든 작업이 완료되면 `.person/reports/{기능명}/report.md` 경로에 작업 결과를 정리한 보고서를 생성합니다.

## 지침 (Instructions)

- **트리거 문구**: "/compound", "컴파운드 실행", "compound 실행", "컴파운드 모드", "compound mode"
- **절대 금지**: 승인 단계 없이 `replace`나 `write_file` 등을 사용하여 코드를 즉시 수정하지 않습니다.
- **보고서 경로**: 반드시 `.person/reports/{기능명}/report.md` 형식을 준수합니다.
